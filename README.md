# Dynamic Financial Executive Summary Using Excel, Co-Pilot, and Claude
I built a dynamic financial summary, KPI dashboard, and AR command center from scratch using Claude and Gemini AI, Microsoft Excel, and Microsoft Co-Pilot.

## 📌 Project Overview
This repository contains the `Nimbus Cloud Solutions_Claude Output.xlsx` project, a comprehensive financial modeling and analytics tool. It demonstrates the ability to translate raw transactional data into high-level executive dashboards, track key performance indicators (KPIs), and maintain rigorous data governance. 

This project highlights core Data Analyst competencies: **Data Aggregation, Dashboard Design, Business Intelligence, and Quality Assurance.**

---
## 1.  AI-Created Fictional Dataset
I used AI to create a new fictional business identity with exact parameters to pair with the excel dynamic template worksheets. 
The data needed to reflect important KPI's regarding financial data for a small to medium size business, and have at least 5,000 records. 
AI created the following:

🏢 New Business Profile

    Company Name: Nimbus Cloud Solutions

    Business Type: B2B Cloud Hosting & Managed IT Services (SaaS)

    Revenue Model: Monthly recurring revenue (MRR) from subscriptions, plus one-time professional services/onboarding fees.

    Financial Profile: High gross margins (typical for SaaS), heavy initial investments in server infrastructure (CapEx), and significant payroll expenses for engineering and sales.

## 📊 2. Executive Dashboards & Visualizations
These tabs serve as the front-end reporting layer, transforming complex data into actionable business insights for stakeholders.

* **KPI Dashboard:** The "executive cockpit." Provides a quick, at-a-glance summary of company health. It highlights a high-growth, high-margin business model that is currently navigating aggressive cash burn (~4 months runway), allowing leadership to make rapid strategic decisions regarding capital and operating expenses.
* **Financial Statements:** The centralized reporting dashboard. Provides a structured, side-by-side view of the Profit & Loss (Income Statement), Balance Sheet, and Cash Flow Statement, complete with period-over-period variance analysis.
* **AR Command Center:** An operational risk control dashboard. It tracks the health of accounts receivable, visualizes aging buckets (0-30, 90+ days), identifies customer concentration risks, and helps prioritize collections to improve liquidity.

## ⚙️ 3. Data Engine & Aggregation
These tabs act as the back-end infrastructure, ensuring accurate calculation, reconciliation, and time-series tracking.

* **Financial Summary:** The engine room of the financial model. It consolidates all financial statements and tracks performance across multiple time horizons (monthly, quarterly, annual, YTD, TTM), serving as the single source of truth for trend analysis.
* **AR Data:** The raw data layer feeding the AR Command Center. It contains granular, invoice-level transactions, enabling precise tracking, aging analysis, and drill-down capabilities from the summary level to specific customer invoices.

## 🛡️ 4. Governance, Validation & Audit
These supporting tabs demonstrate a commitment to data integrity, error-free modeling, and clear documentation.

* **Error Check:** A dedicated quality-control mechanism. It continuously verifies that the model is internally consistent (e.g., ensuring the balance sheet balances and flagging calculation errors), acting as an early warning system for data integrity.
* **Lists:** The behind-the-scenes data dictionary. It centralizes structured reference data (dropdown values, standard categories, mapping tables) to ensure consistency, prevent manual entry errors, and enable clean data validation.
* **PBC Tables (Prepared By Client):** The structured export layer. It organizes the model's complex financial data into clean, flat, audit-ready tables for external reporting and seamless system uploads.
* **AI Log:** A transparent audit trail tracking AI-assisted work. It documents prompts, actions taken, and iterations, ensuring accountability, version control, and clear visibility into the model's development process.
